## 1. プロジェクト概要

ベイズ統計を**中学生レベルの数学から**段階的に学べる大規模学習サイト「ベイズの階段」を作る。
純粋な静的サイト(HTML/CSS/Vanilla JS、ビルド工程なし)で、`index.html` をブラウザで開くだけで動く。

- 対象読者: 中学数学までの知識しかない初学者 〜 実務でベイズモデリングを使いたい人
- 全6レベル + 付録 = **43ページのレッスン** + 7つの目次ページ(うちトップは完成済み)
- 設計思想: 「直感 → 数式」の順。各レッスンは直前のレッスンまでの知識だけを前提にする

## 4. ページ作成手順

### レッスンページ
1. `templates/lesson.html` を `levels/level{N}/{id}.html` にコピー
2. `{{LESSON_ID}}` `{{LESSON_TITLE}}` `{{LEVEL_NUM}}` `{{LEVEL_TITLE}}` `{{DESCRIPTION}}` を置換
3. `<body data-root="../../" data-level="{N}" data-lesson-id="{id}">` を確認(付録は `data-level="appendix"`)
4. `<div class="lesson-body">` 内に本文を書く(テンプレート内のHTMLコメントが構成ガイド)
5. サイドバー・前後ナビ・目次・進捗はJSが自動生成するので**手書きしない**

### レベル目次ページ
1. `templates/level-index.html` を `levels/level{N}/index.html` にコピー
2. プレースホルダを置換(`{{LEVEL_INTRO}}` は2〜4文で新規執筆)
3. レッスン一覧は `#lesson-list` にJSが自動生成する

### 付録ページ
レッスンテンプレートを流用し、`data-root="../"`、CSS/JSパスを `../assets/...` に直す
(付録はルート直下の `appendix/` でレベルより1階層浅い)。パンくずも適宜修正。

## 5. 執筆ガイドライン(全エージェント共通)

### トーンと文体
- です・ます調。講義口調ではなく、隣で教えてくれる先輩の口調
- 1文は短く。中学生が音読してつかえない長さ(目安60字以内)
- 数式を出す**前に**必ず言葉と具体例で説明する(`box-intuition` を使う)
- 専門用語の初出は `<span class="term">用語</span>` で強調し、その場で定義する

### レッスンの標準構成(テンプレートに沿う)
1. **導入**: 身近な問いから始める(2〜3段落)
2. **直感でつかむ**(`box-intuition`)
3. **定義/定理**(`box-def` / `box-thm` + 重要式は `.formula`)
4. **例題**(`box-ex`、必ず完全な解答付き)
5. **理解度チェック**(`.quiz`、1レッスンに1〜3個)
6. **演習**(`box-practice` + `details.answer`、1レッスンに2〜4問)
7. **まとめ**(箇条書き3〜5点)+ 次レッスンへの一言

### 分量・難易度
- 1レッスン = 20〜30分で読める分量(本文3,000〜5,000字程度)
- Level 0〜1: 数式は分数と四則演算まで。表(分割表)と樹形図を多用
- Level 2〜3: Σ・確率記号を使用。微積分が必要な箇所は付録 a-2 へリンク
- Level 4〜5: Pythonコード(PyMC)を含む。コードは `pre > code` で、コピーして動く完全な形で書く

### 数式・図
- 数式は KaTeX: 別行 `$$ ... $$`、行内 `\( ... \)`(テンプレートにCDN設定済み)
- 記号の読み方は初出時にふりがな的に添える(例: 「\(P(A \mid B)\)(「BのもとでのAの確率」と読みます)」)
- 図は inline SVG を `.figure` 内に置く(色は `var(--level-color)` 等のCSS変数を使い、ダークモード対応)
- インタラクティブシミュレーション(`box-sim`)は Vanilla JS + canvas/svg のみ。外部ライブラリ禁止(KaTeX除く)

### 用語の統一(重要)
| 採用 | 使わない |
|---|---|
| 事前確率/事前分布 | プライア |
| 事後確率/事後分布 | ポステリア |
| 尤度(ゆうど) | らしさ(本文の言い換えはOK、用語としては尤度) |
| 信用区間(ベイズ) | 確信区間 |
| 信頼区間(頻度論) | — |

新しい用語を導入したら付録 a-3(用語集)と a-4(記号表)に追記すること。

## 6. 品質チェックリスト(各ページ完成時)

- [ ] `{{...}}` プレースホルダが残っていない
- [ ] `data-root` / `data-level` / `data-lesson-id` が正しい(進捗・ナビが正常動作する条件)
- [ ] ブラウザで開いて: サイドバーに現在レッスンがハイライトされる/前後ナビが正しい/数式が描画される
- [ ] クイズの `data-answer` と選択肢の `data-key` が一致し、解説がある
- [ ] 演習に解答(`details.answer`)が付いている
- [ ] そのレッスンより後のレベルの知識を前提にしていない(**最重要**)
- [ ] ダークモードで図・表が読める(色をハードコードしていない)

## 7. 推奨オーケストレーション

1. **Phase 1 — 骨組み**: 7つの目次ページ(level0〜5 + appendix の index.html)を一括作成。
   この時点で全ナビゲーションが機能し、サイトが「歩ける」状態になる
2. **Phase 2 — 本文**: レベル単位でエージェントに割当(Level 0〜5 は並列可。
   ただし各エージェントは担当レベルより**前の**レベルの完成原稿または curriculum.js の要約を参照し、
   既出の知識・記法に揃えること)。付録 a-1, a-2 は Level 2 と同時か先に作る
3. **Phase 3 — 横断レビュー**: 別エージェントが (a) 前提知識の漏れ(後方参照チェック)、
   (b) 用語・記法の統一、(c) リンク切れ、(d) 用語集/記号表の網羅性 を検査
4. **Phase 4 — 仕上げ**: 付録 a-3〜a-5 を全レッスン確定後に完成させる

並列化の注意: `curriculum.js` は共有ファイルなので、変更が必要な場合は直列化するか Phase 1 で確定させる。

## 8. 技術的注意

- **file:// で動くことが必須要件**。`fetch()` は使わない(カリキュラムを JS グローバル変数にしているのはこのため)
- KaTeX と Google Fonts は CDN 依存(オフラインでは数式が生テキスト表示になるが許容)
- 進捗は `localStorage` キー `bayes-stairs-progress`(完了レッスンIDの配列)、テーマは `bayes-stairs-theme`
- レベルカラーは `main.css` の `--level0`〜`--level5` と `curriculum.js` の `color` の2箇所にある。変更時は両方更新
- 動作確認はブラウザで `index.html` → レベルカード → レッスン → 完了ボタン → トップに戻り進捗反映、の一巡で行う
